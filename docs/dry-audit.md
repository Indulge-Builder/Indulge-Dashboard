# DRY Audit Report тАФ Indulge Live Dashboard

> Senior-principal refactor audit, 2026-06-11. Full read of `components/`, `hooks/`, `lib/`, `app/api/`, `types/`, docs, and `package.json` (~21k LOC, 60 source files).
> **Scope: identify duplication, dead weight, leaks, and doc drift. No code was changed.**
> Hard rule for every proposal below: **zero pixel change, screens stay always-mounted, IST math untouched, ErrorBoundary isolation preserved.**

---

## 0. Executive Summary

The codebase is in better shape than a typical AI-assisted project тАФ `types/index.ts`, `motionPresets.ts`, `GlassPanel`, `StatCard`, `SectionDivider`, `apiGuard`, `webhookGuard` were all built as canonical abstractions. **The core problem: most of them have zero or one consumer.** The abstractions exist; the call sites never migrated. The result:

- **2 dead API routes** (`/api/tickets`, `/api/agents`) that each paginate the *entire* tickets table and are fetched by nobody.
- **5+ hand-rolled stat-tile implementations** next to an unused canonical `StatCard`.
- **The ticket aggregation math implemented twice** (server + client) with a **three-way contradiction in the "Pending" definition** (docs/server say "no date gate", client prune enforces a month gate, the UI label says "This Month").
- **3 webhook routes hand-rolling auth/parse/guard** next to an unused `withWebhookGuard`.
- **4 unused npm dependencies** in the production dependency list.
- **MASTER.md (compiled *today*) disagrees with the code in at least 10 places**, including the agent rosters and the zoho-deals dual-write that no longer exists.
- One genuine multi-day-uptime leak (un-closed `AudioContext` per celebration) and several "hidden screen keeps animating" costs.
